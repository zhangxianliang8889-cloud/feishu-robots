# 机器人数据获取逻辑技术说明文档

## 一、问题分析

### 1.1 问题描述
群消息统计机器人发送的周报显示"0条消息、0人参与"，数据异常。

### 1.2 问题根因

**飞书API时间参数问题**：

飞书消息列表API (`/open-apis/im/v1/messages`) 在使用 `start_time` 和 `end_time` 参数时存在兼容性问题：

| 参数配置 | 返回结果 |
|---------|---------|
| 不带时间参数 | ✅ 返回消息（48条） |
| 带start_time和end_time | ❌ 返回空（0条） |
| 只带start_time | ❌ 报错 |

**根本原因**：飞书API的时间过滤参数在某些场景下不生效，需要在客户端进行时间过滤。

## 三、关键代码片段

### 3.1 消息获取函数（修复后）

```python
def get_messages(token, group_id, start_time, end_time):
    """
    获取群消息（带客户端时间过滤）
    
    Args:
        token: 飞书访问令牌
        group_id: 群ID
        start_time: 开始时间戳（毫秒）
        end_time: 结束时间戳（毫秒）
    
    Returns:
        list: 符合时间范围的消息列表
    """
    url = "https://open.feishu.cn/open-apis/im/v1/messages"
    headers = {"Authorization": f"Bearer {token}"}
    params = {
        "container_id": group_id,
        "container_id_type": "chat",
        "page_size": 50
        # 注意：不使用start_time和end_time参数
        # 原因：飞书API时间参数存在兼容性问题
    }
    
    messages = []
    page_token = None
    
    try:
        while True:
            if page_token:
                params["page_token"] = page_token
            
            resp = requests.get(url, headers=headers, params=params, timeout=10).json()
            
            if resp.get("code") == 0:
                items = resp.get("data", {}).get("items", [])
                
                # 客户端时间过滤
                for msg in items:
                    create_time = int(msg.get("create_time", 0))
                    if start_time <= create_time <= end_time:
                        messages.append(msg)
                
                page_token = resp.get("data", {}).get("page_token")
                if not page_token or len(messages) >= 500:
                    break
            else:
                break
    except Exception as e:
        print(f"获取消息异常: {e}")
    
    return messages
```

### 3.3 用户统计计算

```python
def calculate_user_stats(messages, members):
    """
    计算用户消息统计
    
    Args:
        messages: 消息列表
        members: 群成员映射 {user_id: user_name}
    
    Returns:
        dict: 按消息数降序排列的用户统计
    """
    CEO_LIST = {"张贤良", "蒋文卿"}
    user_stats = {}
    
    for msg in messages:
        sender = msg.get("sender", {})
        sender_id = sender.get("sender_id", {})
        
        # 获取用户ID
        if isinstance(sender_id, dict):
            user_id = sender_id.get("open_id") or sender_id.get("user_id")
        else:
            user_id = sender_id
        
        if user_id:
            user_name = members.get(user_id, "未知用户")
            
            # 过滤CEO和机器人
            if user_name not in CEO_LIST and "机器人" not in user_name:
                user_stats[user_name] = user_stats.get(user_name, 0) + 1
    
    # 按消息数降序排列
    return dict(sorted(user_stats.items(), key=lambda x: -x[1]))
```

## 五、问题排查方向

### 5.1 数据为0的排查步骤

```
1. 检查API响应
   └─▶ code是否为0？
       └─▶ 是：继续检查
       └─▶ 否：检查token和权限

2. 检查消息数量
   └─▶ items是否为空？
       └─▶ 是：群内确实无消息
       └─▶ 否：检查时间过滤逻辑

3. 检查时间过滤
   └─▶ create_time是否在范围内？
       └─▶ 是：消息应该被保留
       └─▶ 否：调整时间区间

4. 检查用户过滤
   └─▶ 是否被CEO过滤？
   └─▶ 是否被机器人过滤？
```

### 5.2 常见问题及解决方案

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| API返回空 | 使用了start_time/end_time参数 | 改用客户端过滤 |
| 时间范围错误 | 时间戳单位错误 | 确保使用毫秒级时间戳 |
| 用户名为空 | 未获取群成员映射 | 先调用获取群成员API |
| CEO未被过滤 | 过滤逻辑未执行 | 检查CEO_LIST配置 |
